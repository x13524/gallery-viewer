# Gallery Viewer

一个运行在浏览器本地的图片 / 视频相册浏览器。它通过 File System Access API 打开本地文件夹，直接浏览本机素材，不上传文件、不依赖后端服务，适合快速查看图片、视频、游戏素材、项目资源和本地截图目录。

## 项目特点

- 本地优先：所有文件都在浏览器本地读取，不上传到云端。
- 媒体浏览：支持图片和视频，适合当作轻量本地相册或素材查看器。
- 深色界面：以媒体内容为视觉主体，弱化后台管理感。
- 大目录友好：支持分批渲染、缩略图懒加载和扫描取消。
- 缓存优化：使用 IndexedDB 缓存缩略图，减少重复生成成本。
- 无构建依赖：HTML、CSS、JavaScript 直接运行，第三方资源本地化。

## 核心功能

- 打开本地文件夹
  - 使用 `showDirectoryPicker()` 授权读取目录。
  - 支持记住上次授权目录。
  - 可选择是否启动时自动恢复上次目录。

- 文件夹导航
  - 左侧文件夹树展示包含媒体的目录。
  - 支持树形视图和平铺视图切换。
  - 支持记住最后选中的文件夹。

- 媒体网格
  - 自适应缩略图网格。
  - 支持小 / 中 / 大三档缩略图尺寸。
  - 支持单击选中、双击预览。
  - 支持图片和视频类型筛选。

- 搜索和排序
  - 按文件名、后缀实时搜索。
  - 支持按文件名、修改日期、文件大小、文件类型排序。
  - 默认按名称排序，减少不必要的文件元数据读取。

- 图片预览
  - 图片弹窗支持滚轮缩放和拖拽平移。
  - 小图按原始尺寸展示，避免被强行放大后发糊。
  - 支持左右方向键切换相邻媒体。

- 视频预览
  - 列表和详情区使用原生 `<video>` 保持轻量。
  - 弹窗预览使用本地 xgplayer。
  - 打开弹窗时会暂停右侧详情视频，避免声音和播放状态冲突。

- 右侧详情
  - 展示当前选中文件的预览图 / 视频。
  - 展示文件名、路径、格式、大小、修改时间、分辨率等信息。
  - 图片存在 EXIF 信息时按需展示。

- 大目录体验
  - 支持渐进式文件列表加载。
  - 支持扫描过程中取消。
  - 支持排除常见工程目录，如 `.git`、`node_modules`、`dist`、`build`。
  - 小目录默认弱化扫描提示，大目录才显示更明显的进度反馈。

- 缩略图缓存
  - 使用 IndexedDB 存储缩略图。
  - 缓存上限为 256MB。
  - 接近上限时提示用户。
  - 支持手动清除缓存。

## 使用方式

推荐使用本地 HTTP 服务打开项目，不建议直接用 `file://` 打开。

启动服务：

```bash
python3 -m http.server 4173
```

打开页面：

```text
http://localhost:4173/index.html
```

停止服务：

```bash
Ctrl + C
```

## 浏览器要求

需要支持 File System Access API 的浏览器。

推荐：

- Chrome
- Edge

说明：

- Safari 对 File System Access API 的支持有限，不建议作为主要运行环境。
- 首次打开目录时需要用户手动授权。
- 浏览器刷新后能否自动恢复目录，取决于浏览器是否仍保留目录读取权限。

## 技术实现

- 页面结构：原生 HTML
- 样式：原生 CSS
- 逻辑：原生 JavaScript
- 文件访问：File System Access API
- 缩略图缓存：IndexedDB
- 用户偏好：localStorage
- 缩略图懒加载：IntersectionObserver
- 视频弹窗播放：xgplayer
- EXIF 解析：exif.js

## 偏好设置

应用会在本地保存部分使用偏好：

- 排序方式
- 排序方向
- 缩略图尺寸
- 媒体类型筛选
- 是否包含子目录
- 是否排除工程目录
- 是否启动时自动恢复上次目录
- 详情面板展开状态
- 最后选中的目录

这些偏好只保存在当前浏览器本地，不会上传。

## 设计取舍

- 只做浏览，不做文件管理。
- 不支持删除、重命名、移动文件。
- 不做云端同步。
- 不做音频播放。
- 不做复杂后台配置。
- 优先保证本地浏览体验、媒体沉浸感和大目录可用性。

## 适用场景

- 浏览本地照片、截图、设计素材。
- 查看游戏 UI、图标、动效和视频素材。
- 快速检查项目资源目录中的图片 / 视频文件。
- 在大型工程目录中筛选和预览媒体资源。

## 注意事项

- 如果目录特别大，首次扫描仍可能需要一定时间。
- 可以开启“排除工程目录”减少 `.git`、`node_modules` 等目录带来的扫描开销。
- 如果误打开了超大目录，可以在扫描过程中取消。
- 如果直接使用 `file://` 打开页面，可能会遇到性能和浏览器安全限制问题，建议始终使用 `localhost` 服务运行。
